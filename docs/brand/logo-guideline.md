# Monetra Logo Guideline

## Purpose

The Monetra logo identifies the product across the website, documentation,
portfolio materials, and application surfaces. Consistent use protects its
clarity and recognizability.

## Logo System

### Primary Mark

Use `monetra-mark.svg` when a standalone brand symbol is appropriate, including
application navigation, profile images, and promotional graphics.

### Horizontal Logo

Use `monetra-logo-horizontal.svg` as the default logo for website headers,
documentation headers, and wide layouts.

### Vertical Logo

Use `monetra-logo-vertical.svg` for centered compositions, portfolio covers,
and layouts where horizontal space is limited.

### Monochrome Mark

Use `monetra-mark-monochrome.svg` when only one color is available, including
watermarks, embossing references, and single-color printing.

### Favicon

Use `monetra-favicon.svg` only for browser tabs and other very small contexts.
Do not use the full horizontal or vertical logo as a favicon.

## Clear Space

Keep clear space around every logo to prevent visual interference.

The minimum clear space is equal to **one quarter of the mark's width**. No
text, icon, border, or other prominent visual element may enter this area.

For horizontal and vertical lockups, measure the clear space from the outermost
edge of the complete logo, including the wordmark.

## Minimum Size

| Asset | Digital minimum |
| --- | ---: |
| Favicon | 16 × 16 px |
| Standalone mark | 32 × 32 px |
| Horizontal logo | 160 px wide |
| Vertical logo | 120 px wide |
| Monochrome mark | 24 × 24 px |

Use the favicon instead of the full mark when the available size is smaller
than 32 × 32 pixels.

## Background Usage

### Light Backgrounds

Use `monetra-logo-on-light.svg` on white, off-white, or other light neutral
surfaces.

### Dark Backgrounds

Use `monetra-logo-on-dark.svg` on dark green, charcoal, black, or other dark
surfaces.

### Transparent Mark

Use `monetra-mark-transparent.svg` when the mark must appear without a
containing background. Ensure sufficient contrast between every part of the
mark and the underlying surface.

### Photography and Illustrations

Place the logo only over quiet areas with sufficient contrast. When the
background is visually complex, use the primary mark with its containing shape
or place the logo on a solid neutral panel.

## Color Integrity

Use only the approved brand assets and colors. Do not recolor individual parts
of the logo or create unapproved color combinations.

The monochrome mark may be rendered in:

- `#111814` on light backgrounds.
- `#F8FAF9` on dark backgrounds.

## Wordmark

The wordmark is always written in lowercase:

> monetra

Do not recreate the wordmark by typing it separately. Use the provided
horizontal or vertical logo asset to preserve spacing and proportions.

## App Icon

`monetra-app-icon.png` is the detailed 3D application icon. Use it for:

- Portfolio presentations.
- Application installation artwork.
- Social preview graphics when appropriate.
- Promotional mockups.

Do not use the app icon as a replacement for the flat logo in navigation,
forms, small controls, or text-heavy layouts.

## Incorrect Usage

Do not:

- Stretch, compress, rotate, skew, or distort the logo.
- Rearrange the mark and wordmark.
- Change the approved colors.
- Add shadows, glows, outlines, textures, or gradients.
- Place the logo over a low-contrast background.
- Crop any part of the logo.
- Add text inside the required clear space.
- Use the detailed app icon at small interface sizes.
- Animate individual logo parts without an approved motion specification.

## File Selection

| Context | Preferred asset |
| --- | --- |
| Website header | `monetra-logo-horizontal.svg` |
| Mobile navigation | `monetra-mark.svg` |
| Browser tab | `monetra-favicon.svg` |
| Light background | `monetra-logo-on-light.svg` |
| Dark background | `monetra-logo-on-dark.svg` |
| Single-color output | `monetra-mark-monochrome.svg` |
| Transparent composition | `monetra-mark-transparent.svg` |
| Portfolio app showcase | `monetra-app-icon.png` |

## Accessibility

- Provide the accessible name **Monetra** when the logo acts as a link.
- Use empty alternative text when adjacent text already identifies Monetra.
- Decorative logo instances should be hidden from assistive technology.
- Do not rely on the logo alone to communicate application state or actions.
